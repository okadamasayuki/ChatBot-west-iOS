# 会計相談チャット (iOS)

[chatbot-west](https://github.com/okadamasayuki/chatbot-west)(Web版)から派生した iOS ネイティブ版です。SwiftUI 製で、**Web版と同じ Firebase(Auth / Firestore / Functions)に接続**するため、相談・案件・社内ルール・マニュアルのデータは Web版と共有されます。ただし機能は iOS 版で独自に進化しており、Web版とは差分があります(下記「Web版との違い」参照)。

## 画面構成(タブ)

| タブ | 対象 | 内容 |
|---|---|---|
| 質問者 / 相談一覧 | 全員 | 相談チャットの一覧と LINE 風チャット画面。バッジ = 財務は未対応の案件数、質問者は未読の返答数 |
| BAチャット | 財務のみ | BA 同士のトーク(1対1・グループ)。バッジ = 全トークの未読合計 |
| 通知 | 財務のみ | アプリ内通知の一覧。バッジ = 未読通知数 |
| マニュアル | 財務のみ | 社内マニュアルの管理(PDF / テキスト / Markdown) |
| 設定 | 全員 | アカウント・メンバー管理・社内ルール・回答の癖・Q&A エクスポートなど |

## 主な機能

### 相談チャット
- 💬 **LINE風チャットUI + トークルーム** — 相談を質問ごとのルームに分けて管理。「＋ 新規」で作成、スワイプで削除(自分の相談のみ)
- 🔀 **並び替え** — 新しい順 / ステータス順(BA回答待ち → 担当者回答待ち → 完了)
- 🏷 **ステータスタグ** — 担当BAが未割り当てで AI も未回答なら「BA回答待ち」、以降は担当・回答状況に応じて自動で変化
- 🤖 **Claude API による自動回答** — 一般的な会計知識で答えられる質問は AI が回答。情報不足のときは短い聞き返し+選択肢ボタンを表示
- 👤 **相談ごとの担当BA** — 案件カードはチャット内に統合。担当BAの割り当て・引き継ぎはチャット画面のヘッダーから行う
- 🤝 **対応依頼の承諾フロー** — 他のBAへの対応依頼は相手の**承諾が必要**。依頼された側にはチャット上部にバナー(承諾 / 辞退)が表示され、結果は依頼した側に通知される
- ✏️ **メッセージの編集・削除・コピー・リアクション** — 長押しメニューから。削除はソフトデリートで復元可能
- 📝 **要約** — これまでのやり取りを AI が要約表示
- ✅ **相談の完了** — 完了した相談は案件・バッジの集計から除外
- 🔗 **相談リンク** — 相談へのリンクをコピーして BA チャットに貼るとリンクカードになり、タップでその相談へ移動

### BAチャット・通知・未読
- 👥 **BAチャット** — BA 同士のトーク。スワイプで削除、未読数はトークごとに緑バッジで表示
- 🔔 **アプリ内通知**(サーバーレス) — 新着メッセージ・対応依頼・承諾/辞退結果を各端末が Firestore の変化から検知して蓄積。アプリを閉じている間の新着も次回起動時にキャッチアップ
- 📩 **通知の操作** — タップで該当チャットの最下部へ移動、左スワイプで削除、右スワイプで未読⇄既読。未読の通知は緑の太字で表示
- 🔢 **未読バッジ** — 相談一覧・BAチャット一覧・各タブに LINE 式の未読数を表示。既読は `reads.{uid}` で端末間同期

### マニュアル・社内ルール ほか
- 📚 **社内マニュアル** — PDF / テキスト / Markdown を追加(PDF は文字抽出+プレビュー)。回答時に AI が参照し、**該当箇所を PDF 上でハイライト表示**
- 📘 **社内ルール** — 回答時に AI が最優先で参照。相談やマニュアルから差分を抽出して追記、AI で整理してコンパクト化も可能(設定タブから)
- ✍️ **回答の癖(回答スタイル)** — アカウントごとに回答文の書き方を登録
- 🎤 **音声入力** — 回答方針を日本語音声で入力(Speech / マイク権限が必要)
- ⬇️ **Q&A履歴のエクスポート** — JSON / CSV(BOM付きUTF-8・Excel対応)を共有シートから書き出し
- 🧪 **開発モード** — AI が「会計素人の担当者役」を演じて返答・更問し、納得したらお礼と完了まで行う動作確認用モード
- 🔐 **メール+パスワードのログイン** — 新規登録時にニックネームと役割(担当者 / 財務=BA)を選択

## ビルド方法

1. [XcodeGen](https://github.com/yonaskolb/XcodeGen) でプロジェクトを生成(生成済みの `.xcodeproj` があればそのままでもOK):
   ```sh
   brew install xcodegen
   xcodegen generate
   ```
2. `ChatBotWest.xcodeproj` を Xcode で開く(初回は Firebase SDK を Swift Package Manager が自動解決)
3. Signing & Capabilities で自分の開発チームを選択
4. iPhone(シミュレータ or 実機)で Run

## Firebase 接続について

- Web版と同じ Firebase プロジェクト(`chatbotwest-5b568`)・同じワークスペース合言葉を埋め込みで使用しています(`ChatBotWest/App.swift`)
- `GoogleService-Info.plist` をターゲットに追加すると、そちらの設定が優先されます(Firebase コンソールで iOS アプリを登録する場合)
- Anthropic API キーはサーバー(Firebase Functions の `chat`)側で管理されるため、通常は端末に不要です。サーバー未設定時のみ、設定タブのAPIキー(フォールバック)が使われます

## 構成

```
project.yml            … XcodeGen プロジェクト定義(Firebase SPM 依存を含む)
ChatBotWest/
  App.swift            … エントリポイント + Firebase 初期化
  Models.swift         … Room / Message / Case / BaTalk / Manual / QaEntry(Firestore と同一構造)
  Prompts.swift        … トリアージ・回答案・社内ルール抽出などのプロンプト
  ClaudeService.swift  … Functions プロキシ経由の Claude 呼び出し(直接呼び出しフォールバック付き)
  CloudStore.swift     … Firestore 同期・アプリ状態・通知/未読の検知・業務ロジック
  SampleData.swift     … デモ用サンプル相談・サンプルBAチャット・サンプルマニュアル
  Views/
    RootView.swift          … タブ構成
    ChatTab.swift           … 相談一覧(未読バッジ・ステータスタグ・並び替え)
    ChatRoomView.swift      … 相談チャット(担当・承諾バナー・要約・編集/削除)
    BaChatView.swift        … BAチャット(トーク一覧・トーク画面・リンクカード)
    NotificationsView.swift … 通知一覧(スワイプで削除/未読⇄既読)
    NaikiManualViews.swift  … 社内ルール・マニュアル
    SettingsView.swift      … 設定(共通スワイプ行 SwipeDeleteRow もここ)
    ほか ログイン / メンバー / PDF・添付 / 音声入力 / テーマ
```

## Web版との違い

データは Firestore の `workspaces/{wid}/…`(rooms / cases / qa / manuals / members / baTalks)を Web版と共有しますが、機能は iOS 版で独自に進化しています。

| Web版 | iOS版 |
|---|---|
| BAタブ(案件一覧) | **廃止** — 案件カードは相談チャット内に統合 |
| 案件単位の「要対応」 | 相談ごとの担当BA(`rooms.handler`)+ 対応依頼の承諾フロー |
| 通知なし | アプリ内通知タブ + LINE式未読バッジ |
| — | メッセージ編集 / 削除(復元可) / リアクション / 要約 / PDFハイライト / 開発モード |
| 社内ルールタブ | 設定タブ内に移動 |
| Web Speech API の音声入力 | SFSpeechRecognizer(ja-JP) |
